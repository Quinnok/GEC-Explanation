# Claim-Evidence Matrix

Last updated: 2026-07-18

| Claim | Current Evidence | Status | Risk |
|---|---|---|---|
| The project can run on real English GEC data. | EXPECT MIT-licensed sample: 300 source/reference pairs and 320 ERRANT edits in `data/processed/expect_v1_samples.jsonl`. | Supported for pilot | Full benchmark coverage still pending. |
| ERRANT is preferable to token diff for typed edit extraction. | Token diff matches ERRANT exactly on 94.7% of sampled sentences before type labels. | Supported for pilot | ERRANT itself can have boundary ambiguity. |
| The project now has real GEC model predictions, not reference-as-prediction placeholders. | `results/model_predictions/expect_v1_model_predictions.jsonl` has 600 rows from GECToR and T5 over 300 EXPECT sources; runtime metadata records versions and decoding. | Supported for pilot | GECToR model card is non-commercial; T5 model card is CC-BY-NC-SA-4.0. |
| Model-produced edit data can be constructed at edit level. | `results/model_edits/model_edit_dataset.jsonl` contains 1707 actual predicted edits; `missing_edit_diagnosis.jsonl` contains 319 missed reference edits. | Supported for pilot | ERRANT and alignment noise remain, especially with detokenized seq2seq outputs. |
| GEC model behavior can be classified into correct, wrong, overcorrected, and missed corrections. | Behavior summary: GECToR correct 142, wrong 37, over 493, missed 141; T5 correct 71, wrong 71, over 893, missed 178. | Supported for pilot | Labels are automatic alignments, not human adjudications. |
| Explicit explanations can leak the answer to a structured reconstructor. | Round 02 structured baseline explicit Full Edit Exact: 99.7%; raw edit string Full Edit Exact: 100.0%. | Supported only as leakage upper control | This must not be claimed as reverse reconstruction effectiveness. |
| Masking the target reduces target reconstruction in the template pilot. | Round 02 structured baseline masked-target Target Match: 11.7%. | Supported only as leakage diagnostic | Masked templates are automatic, not human explanations. |
| EXPECT contains useful explanation-adjacent labels but no direct natural-language explanations. | Local EXPECT README and JSONL fields expose `evidence_index` and `error_type`; no free-text explanation field is present. | Supported | Need a license-clear human NL explanation source or annotation for final claims. |
| Open-source explanation candidates can be generated without paid APIs. | `data/processed/model_edit_explanation_candidates.jsonl` has 300 FLAN-T5-base candidates from source, model prediction, and predicted edit span. | Supported as candidate generation only | Candidate quality is weak; not human gold and not yet a main experiment explanation source. |
| The paper's claimed contribution remains distinct from closest work. | Round 03 literature pass found related GEC explanation generation/evaluation and simulatability work, but no verified edit-wise GEC reverse reconstruction evaluation. | Plausible but not final | Do not use "first" or "novel" until a final systematic pass is complete. |
| T5 raw behavior is substantially affected by detokenization/formatting edits. | Round 04 normalization ablation: raw T5 predicted edits 1035 with precision 0.068599; excluding ORTH+PUNCT predicted edits 465 with precision 0.152688; substantive-only predicted edits 361 with precision 0.196676. | Supported for pilot | Normalization may also remove meaningful orthographic corrections; raw stats must remain visible. |
| Exact-only edit alignment overstates over/missed behavior. | Round 04 comparison over 600 sentences: exact-only over 1494 and missed 427; stable partial-overlap alignment over 1404, missed 337, wrong 90. | Supported for pilot | Stable alignment requires manual validation on ambiguous cases. |
| Reverse Edit Reconstruction is not sufficient as the paper's main novelty. | Round 02 showed explicit/raw template leakage; Round 05 literature package identifies strong prior work on simulatability, GEC explanation generation, edit-wise EXCGEC, CLEME2.0 behavior decomposition, GEC counterfactual robustness, and self-consistency critiques. | Supported for framing | Needs formal Round 06 idea scoring and rewritten paper framing. |
| Counterfactual Edit Simulatability is the selected main line. | Round 06 compared six candidate ideas, simulated five reviewer perspectives, selected Counterfactual Edit Simulatability as main, and selected Rule-grounded Faithfulness as backup. | Supported for planning, not empirically supported | Round 08/09 must implement counterfactuals and rerun GEC models; COCOGEC is a close threat. |
| Rule-grounded Faithfulness is the backup line and L3 component. | Round 06 scoring rated it high in GEC specificity and feasibility, especially for learner-facing explanations and EXPECT/ERRANT labels. | Supported for planning | It may measure grammatical validity or plausibility rather than model faithfulness unless tied to behavior. |
| The project now has a model-edit-level benchmark candidate rather than only a source-reference template pilot. | `data/faithfulness_benchmark/edit_records.jsonl` contains 700 selected model-produced edits from EXPECT and JFLEG; `explanation_instances.jsonl` contains 12,754 automatic explanation/control instances; `missing_edit_diagnosis.jsonl` contains 160 missed-edit diagnoses. | Supported for automatic benchmark construction | Labels are automatic and template-heavy; no human gold labels exist yet. |
| The benchmark covers at least two datasets and three model families at pilot scale. | Round 07 selected counts: EXPECT 420, JFLEG 280; GECToR 298, T5 280, CoEdIT 122; model-family counts include sequence-to-edit, sequence-to-sequence, and instruction-following text editor. | Supported for pilot | CoEdIT evidence is only 20 EXPECT sources; JFLEG uses `ref0` as the primary reference for ERRANT alignment. |
| Explicit and masked templates should be treated as leakage controls, not primary explanations. | `docs/benchmark_card.md` and `data/faithfulness_benchmark/leakage_audit.json` record 700 explicit templates, 700 masked-target templates, high template near-duplicate risk, and 0 human-gold labels. | Supported for framing | Primary experiments must report stratified metrics and avoid template-dominated macro averages. |
| Human faithfulness correlation is not yet established. | No double-human annotation has been collected; Round 07 `human_gold_count` is 0. | Unsupported | Round 10 must prepare annotation packets and collect or request real human judgments before any human-alignment claim. |
| L1 reconstruction is sensitive to answer leakage. | Round 08 reconstruction full edit exact: structured/reverse 0.098, target-masked 0.015, leakage-adjusted 0.035; classification macro-F1 drops from reverse 0.640 to target-masked 0.439 and leakage-adjusted 0.496. | Supported for automatic benchmark pilot | Because labels/templates are automatic, this supports leakage diagnosis rather than final explanation faithfulness. |
| Nontrivial automatic baselines can be run over the benchmark. | Round 08 ran surface keyword, structured extraction, TF-IDF similarity, NLI lexical proxy, rule/evidence verifier, and ablations over 11,764 labeled instances. | Supported for implementation | Strong rule/evidence numbers may exploit automatic template artifacts. |
| Counterfactual labels can be obtained from actual model reruns. | Round 08 built 48 counterfactual variants and reran GECToR, T5, and CoEdIT; actual labels were preserve 23, competing_edit 20, cancel 3, change_span 2. | Supported for small pilot | Pilot size is small; counterfactual validity and competing-edit interpretation need Round 09 scaling and audit. |
| The current L2 simulator baselines are weak and mainly diagnostic. | Round 08 counterfactual macro-F1: random 0.150, source-edit availability 0.161, variant-family prior 0.193. | Supported as baseline result | Stronger explanation-conditioned simulator remains to be implemented and compared. |
| L1 leakage diagnosis is statistically stable under grouped bootstrap. | Round 09 grouped bootstrap macro-F1: reverse reconstruction 0.640 [0.633, 0.646], target-masked reconstruction 0.439 [0.428, 0.451], leakage-adjusted reconstruction 0.496 [0.489, 0.501]. | Supported for automatic benchmark pilot | This remains a leakage/control finding, not a human faithfulness result. |
| Rule/evidence verifier is the strongest automatic L1/L3 baseline in the current synthetic benchmark. | Round 09 grouped bootstrap macro-F1: rule/evidence verifier 0.767 [0.752, 0.785], NLI proxy 0.736 [0.731, 0.740], TF-IDF similarity 0.703 [0.694, 0.714]. | Supported for automatic benchmark pilot | The verifier may exploit synthetic rule/evidence construction; human validation is required. |
| Round 09 does not support a nontrivial counterfactual simulator gain. | Scaled L2 pilot has 120 counterfactual variants and 1,080 explanation-variant pairs; best simulator macro-F1 is trained explanation-type prior 0.297 [0.277, 0.312], with explanation leakage simulator 0.222 [0.195, 0.244]. | Supported as negative result | Need stronger simulator or narrower paper claim. |
| Rule-relevant variants are disruptive but often not clean cancellations. | Round 09 actual labels: preserve 57, competing_edit 52, cancel 7, change_span 3, change_target 1. Rule-relevant variants include 47 competing edits and only 7 cancels in the baseline summary. | Supported for pilot | Counterfactual validity filtering and competing-edit analysis are required before paper claims. |
