\begin{abstract}
Grammatical error correction (GEC) systems increasingly provide natural-language explanations for their corrections, but a fluent explanation may fail to describe the edit that the model actually made or the rule and evidence that would justify it. We study edit-level explanation evaluation for model-produced English GEC edits and separate edit alignment, edit validity, rule correctness, evidence grounding, and overall faithfulness. Our executable benchmark contains 700 model edits from EXPECT and JFLEG, 12,754 automatic explanation/control instances, and 120 counterfactual variants labeled by rerunning the original GEC models. We further evaluate 160 edit-explanation stress-test items with two independent human annotators and a human adjudicator. Reverse reconstruction is useful for edit alignment, but it remains weak on rule and evidence dimensions: against adjudicated human labels it obtains 0.789 binary faithfulness Macro-F1 but only 0.431 rule Macro-F1 and 0.426 evidence Macro-F1. A rule/evidence verifier improves rule recall but remains too noisy to support a solved RuleFaith claim. These findings support a cautionary conclusion: edit reconstruction is necessary but insufficient for evaluating GEC explanations, and automatic explanation metrics should be stress-tested against rule and evidence judgments rather than optimized for edit copying.
\end{abstract}
