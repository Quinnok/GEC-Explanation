\section{Introduction}

Grammatical error correction (GEC) systems transform learner or user-written text into corrected text while preserving the intended meaning. As these systems move into writing assistance and language-learning settings, the correction alone is often insufficient: users also need to understand what changed, whether the change is appropriate, and which contextual evidence supports the correction. Recent explainable GEC work has therefore introduced evidence words, error types, and natural-language explanations for corrections \cite{fei-etal-2023-enhancing,song-etal-2024-gee,kaneko-okazaki-2024-controlled,ye2025excgec}.

The central difficulty is that explanation fluency is not the same as explanation faithfulness. A generated explanation may accurately restate that a model replaced one token with another while giving the wrong grammar rule. It may cite subject--verb agreement for an article edit, point to the wrong contextual evidence, or present an invalid model edit as if it were grammatically required. Conversely, an explanation can faithfully describe a wrong model edit if it clearly states that the edit is questionable. For GEC, these distinctions matter because learners may use explanations to decide whether to accept a correction and how to revise future sentences.

Prior diagnostic work in this project showed why edit reconstruction is useful but incomplete. A reconstructor can test whether an explanation contains enough information to recover the observed edit, but this test is leakage-prone when the explanation directly copies the source and target strings. On a human-adjudicated stress test, reverse reconstruction reaches strong coarse faithfulness and edit-alignment scores, yet remains weak on rule and evidence dimensions. This motivates a method that does more than optimize for reconstructability.

We propose RuleFaith-GEC, a verifier-guided generation, repair, selection, and abstention framework for edit-level GEC explanations. Given a source sentence, a model prediction, and one atomic model-produced edit, RuleFaith first generates multiple explanation candidates. It then verifies separate dimensions: edit alignment, edit validity, rule plausibility, evidence sufficiency, specificity, target leakage, and genericness. Failed dimensions are converted into targeted repair instructions. The repaired candidate is revalidated under target-masked controls before the system selects an explanation or abstains when the candidates remain unreliable.

This framing keeps the paper's claim modest. RuleFaith does not infer the hidden causal mechanism of a neural GEC model. It evaluates and improves behavioral consistency with the observable edit, grammatical validity, and source-grounded evidence. Reverse reconstruction remains one component for edit alignment; it is not treated as a complete faithfulness metric.

Our current evidence has two roles. First, a 160-item human-adjudicated stress test establishes that reconstruction-oriented metrics do not adequately capture rule and evidence correctness. Second, a Qwen3-8B local-teacher method pilot exercises the RuleFaith loop on natural teacher candidates: field-aware selection, target-masked validation, rule/evidence audit, targeted repair, and a Codex/AI pseudo-validation pass. The pseudo-validation is used only for internal triage and is not reported as human gold.

The contributions are:

\begin{itemize}
\item We formulate faithful edit-level GEC explanation generation as a multi-dimensional verifier-guided process rather than a single reconstruction or LLM-judge score.
\item We implement RuleFaith-GEC, which converts failed verification dimensions into targeted repairs and performs leakage-aware revalidation before selecting or abstaining from explanations.
\item We provide human-grounded diagnostic evidence and an open Qwen3-8B method pilot showing both the value and the limits of automatic RuleFaith gates.
\end{itemize}
