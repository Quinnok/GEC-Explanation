\section{Conclusion}

This paper formulates edit-level explanation evaluation for GEC as a three-layer problem: edit correspondence, counterfactual edit simulatability, and rule/evidence grounding. The automatic pilot builds a model-produced edit benchmark, runs real GEC model predictions, extracts edits with ERRANT, reruns models on counterfactual variants, and tests leakage controls, baselines, and reranking. The main lesson is cautionary: reconstruction and reranking can look strong when explanations copy edit fields, while counterfactual behavior remains difficult to simulate. A submission-ready version therefore needs real human annotation and stronger counterfactual validation before making claims about human faithfulness or learner helpfulness.
