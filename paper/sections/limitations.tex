\section{Limitations}

This pilot uses many automatic template explanations constructed from ERRANT edit fields. They are useful for testing leakage, reconstruction behavior, and negative controls, but they should not be interpreted as learner-facing explanation quality. The current human-adjudicated set is intentionally stress-test-oriented rather than a natural random sample of GEC explanations. The Qwen3-8B method pilot uses natural teacher candidates, but its final ready-candidate validation is Codex/AI pseudo-validation, not human gold. It can guide internal repair and selection decisions, but final method claims require a blinded natural-explanation human evaluation. The structured baseline is intentionally narrow: it detects explicit edit statements, so strong performance on explicit and raw-edit conditions does not imply that free-form explanations are faithfully understood. ERRANT edit boundaries and error types may differ from valid alternative corrections. Finally, the current implementation does not yet include student distillation, preference optimization, or a complete risk--coverage evaluation; those remain required before claiming a full deployable RuleFaith-GEC system.
